# Session Journal -- 2026-03-16

## Summary

v1 production readiness pass. 10 issues fixed across backend and frontend in 3 commits.

## What Was Done

### Tier 1 -- Critical

**GET /playlists OOM (`backend/src/routes/playlists.ts`)**
- Root cause: Prisma `include` was loading full track+album+artist tree for all items in all playlists in one query. With 50 playlists x 500 tracks each, that's 25k tracks loaded on every list view.
- Fix: replaced deep include with `_count: { select: { items: true } }` + a 4-item mosaic select (only `coverUrl` per track). Playlist list now returns O(n_playlists) rows, not O(n_playlists * n_tracks).
- Bonus fix: discovered pre-existing `coverArt` vs `coverUrl` field mismatch in `PlaylistMosaic` component -- mosaic was always showing empty state. Fixed in same change.

**recently-listened DoS (`backend/src/routes/library.ts`)**
- `limit` query param had no upper bound -- could request unlimited rows.
- Fix: `Math.min(parseInt(...) || 10, 100)`.

**N+1 Deezer API calls (`backend/src/routes/library.ts`)**
- Three separate `Promise.all(...map(async...))` loops were hitting Deezer with no concurrency cap. With 20+ similar artists, fires 20+ simultaneous HTTP requests.
- Fix: `pLimit(3)` wrapping each loop. `p-limit` was already installed (^7.2.0, CJS compatible).

**SSRF in systemSettings (`backend/src/routes/systemSettings.ts`)**
- `test-lidarr`, `lidarr-profiles`, `test-audiobookshelf` passed user-supplied URLs directly to axios.
- Fix: `validateUrlForFetch` from `backend/src/utils/ssrf.ts` applied before each axios call.

**requireAdmin double-auth (`backend/src/routes/systemSettings.ts`)**
- Router had both `requireAuth` then `requireAdmin` -- two JWT verifications + two DB lookups per request. `requireAdmin` already calls `authenticateRequest` internally.
- Fix: removed `router.use(requireAuth)`.

**webhookEventStore.test.ts (`backend/src/services/__tests__/`)**
- Test was hitting real PostgreSQL (not running in CI/test environments) -- failing with `PrismaClientInitializationError`.
- Fix: complete rewrite as mocked unit test matching the auth/playlists route test pattern. All 10 tests pass.

### Tier 2+3 -- High

**UMAP nNeighbors scaling (`backend/src/services/umapProjection.ts`)**
- Hard cap of 15 caused map fragmentation for libraries > ~200 tracks.
- Fix: `Math.min(50, Math.max(5, Math.round(Math.sqrt(rows.length))))` -- 100 tracks -> 10, 500 -> 22, 1000+ -> 32-50.

**Circular layout cache TTL (`backend/src/services/umapProjection.ts`)**
- Small libraries (<5 tracks) get circular layout cached for 24h -- even after enrichment runs and UMAP becomes possible, old circular layout stays.
- Fix: `CIRCULAR_CACHE_TTL = 3600` (1h). `cacheResult` accepts optional `ttl` param.

**Timer cleanup on unmount (VibeMap.tsx, vibe/page.tsx)**
- `saveTimerRef` and `searchTimerRef` `setTimeout` refs had no cleanup -- could fire after unmount and update stale state.
- Fix: `useEffect(() => () => { if (ref.current) clearTimeout(ref.current); }, [])` in both files.

## Decisions Made

- `validateUrlForFetch` from `ssrf.ts` is the canonical SSRF defense for all endpoints accepting user URLs -- apply consistently, don't roll your own checks.
- `requireAdmin` already authenticates -- never pair with `requireAuth` on the same router.
- `pLimit(3)` is the standard concurrency cap for outbound API calls.

## Surprises

- `useEffect` was missing from the React import in VibeMap.tsx -- `useCallback, useMemo, useRef, useState` but no `useEffect`.
- webhookEventStore test had been untested in CI the entire time -- would have caught the missing mock on first run.
- Mosaic bug (coverArt vs coverUrl) was pre-existing and silently broken -- users never saw album art in playlist mosaics.
